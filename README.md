# ブロック崩し (raylib)

## 実行方法
- 依存: raylib がインストール済みであること
- ビルド:
```
make
```
- 実行:
```
./pong
```
- 実行時に `NotoSansMono-Regular.ttf` と `*.wav` が同じディレクトリに必要

## 操作方法
- 移動: 左右キー / A・D
- 発射: Space
- ポーズ: P
- レベル選択: 1/2/3 または ↑↓、クリック/Enter で開始
- CLEAR/OVER 画面: Enter で進行/メニューへ

## 機能
- ゲーム状態: MENU / PLAY / PAUSE / CLEAR / OVER
- レベル: EASY / NORMAL / HARD（HARD は壊せないブロックあり）
- クリア条件: 壊せるブロックをすべて破壊
- ブロック種別:
  - 通常ブロック: 白色
  - 機能ブロック: 色付き、アイテムを落とす
  - 壊せないブロック: 濃い灰色
- アイテム（落下）:
  - M: マルチボール（最大2個追加）
  - E: パドル幅拡張（リセットまで持続）
  - S: スローボール（一定時間）
  - F: 高速ボール（一定時間）
  - L: 残機追加
  - X: 残機が1つ減る
- 速度管理: `speed_state` と `speed_timer` で低速/通常/高速を制御
- 演出: BGM/効果音、パーティクル、画面揺れ

## 工夫したところ
- ボールがパッド上にある間はパッド移動とアイテム落下を停止し、開始タイミングを揃えた
- パドルの当たり位置で反射角度が変わるようにし、狙い撃ちしやすくした
- ブロック破壊時にパーティクルと画面揺れを入れて打感を強化
- メニューにボタン型 UI を用意し、キーボードとクリックの両方に対応
- 速度変化はタイマーで自動復帰するようにして管理を簡潔化
- 授業で扱った構文を活用して可読性と管理性を向上
- typedef+struct を併用し、Ball/Brick/Particle などの関連データを1つの型にまとめて記述を簡潔化
- enum を使って GameState/PowerType を定義し、マジックナンバーを排除して意図が伝わる条件分岐に整理
- ポインタ渡し（例: InitLevel の breakable_left、LaunchBall の Ball*）で関数内から状態を更新し、コピーを避けて効率化

## 課題点
- 効果音が同じファイルで代用されており、種類ごとの音の差が弱い
- 速度変化の残り時間や効果中の表示がなく、状態が分かりにくい
- マルチボール時の視認性や情報量が増えるため、UI の整理が必要

## 使っている素材
- `background.wav`, `gameclear.wav`, `gameover.wav`（効果音/音楽）
- `NotoSansMono-Regular.ttf`（フォント）
